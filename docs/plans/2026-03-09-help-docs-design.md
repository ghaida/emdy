# Help Docs Design

## Overview

Add help documentation to Emdy through three pieces: a native macOS Help Book for detailed docs, an in-app keyboard shortcut cheat sheet, and a first-launch welcome screen.

## 1. macOS Help Book

A standard Apple Help Book bundle registered via `Info.plist` keys (`CFBundleHelpBookFolder`, `CFBundleHelpBookName`). Opens in Help Viewer from the Help menu.

### File structure

```
Emdy/Resources/Emdy.help/
  Contents/
    Info.plist
    Resources/
      en.lproj/
        index.html        # Landing page with links to all topics
        opening.html      # Opening files: single, directory, drag-drop, Open Recent
        navigating.html   # Heading navigator, minimap, find in page, anchor links
        display.html      # Zoom, font styles, themes
        sharing.html      # Copy as RTF, export PDF, print
        shortcuts.html    # Full keyboard shortcut reference
      search.helpindex    # Generated by hiutil for searchability
```

### Content topics

- **Opening files** — single file, directory browsing, drag-and-drop onto Dock icon/window, Open Recent
- **Reading & navigating** — heading navigator sidebar, minimap, find in page (Cmd+F), anchor links
- **Display settings** — zoom in/out/reset, three font styles (serif, sans-serif, monospace), three themes (light, dark, system)
- **Sharing** — copy selected text as RTF, export as PDF, print
- **Keyboard shortcuts** — full reference table

Styling is minimal — Apple Help Viewer handles most presentation. Pages use clean HTML with light CSS.

## 2. Keyboard Shortcut Cheat Sheet

A `ShortcutCheatSheet` SwiftUI view presented as a `.sheet`, accessible from Help > Keyboard Shortcuts.

### Shortcut categories

- **File** — Open, Open Folder, Close, Print, Export PDF
- **View** — Zoom In/Out/Reset, Font (1/2/3), Theme (7/8/9), Toggle Minimap, Toggle Headings
- **Edit** — Copy, Find, Select All

### Layout

Each row shows the action name on the left and a shortcut key badge on the right. Grouped by category with section headers. Styled to match the app's current theme using `ColorPalette`.

## 3. Welcome Screen

A `WelcomeView` SwiftUI sheet shown on first launch. Accessible again from Help > Welcome to Emdy.

### First-launch tracking

`UserDefaults.standard.bool(forKey: "hasShownWelcome")` — set to `true` after first display.

### Content — four feature highlights

1. **Read Markdown beautifully** — serif, sans-serif, and monospace rendering
2. **Navigate with ease** — heading navigator and minimap
3. **Make it yours** — themes, zoom, and font switching
4. **Share your way** — copy RTF, PDF export, print

### Actions

Bottom of the sheet: "Open File" and "Open Folder" buttons plus a "Get Started" dismiss button. No "don't show again" toggle needed since it only appears on first launch.

## Integration points

- **Help menu**: Add "Keyboard Shortcuts" and "Welcome to Emdy" menu items in `EmdyMenuCommands.swift`
- **Info.plist / project.yml**: Register Help Book keys
- **EmdyApp.swift**: Check `hasShownWelcome` on launch and present `WelcomeView`
