On reading

Every day, more of what we read arrives as Markdown. Design specs, product briefs, technical documentation, meeting notes generated by AI — all of it written in a format designed for machines to parse, not for people to enjoy.

We built Emdy because reading deserves the same care we give to writing. A well-rendered page with considered typography, proper spacing, and beautiful syntax highlighting transforms a wall of markup into something you actually want to spend time with. The way a good book makes you slow down and pay attention.

What Emdy does

A reader that treats every Markdown file with the care of a printed page — because the way something looks changes how it feels to read.

i.

Beautiful rendering

Full GitHub Flavored Markdown with syntax-highlighted code blocks, properly styled tables, and typography that follows a strict 4px grid. Three font families — sans, serif, and mono — all from the IBM Plex family, all beautifully set.

ii.

Navigate with ease

Open a folder and browse every Markdown file from a clean sidebar. A canvas-based minimap gives you the shape of the whole document at a glance. Command palette search finds anything across file names and content.

iii.

Your fonts, your way

Switch between sans, serif, and monospaced typefaces. Zoom in and out with familiar shortcuts. Choose between two color themes — a warm palette inspired by Dieter Rams and a cool slate blue — each with light, dark, and system-matched appearances.

iv.

Share what you read

Export to PDF, copy as beautifully formatted HTML, or print directly. When someone else updates the file, Emdy notices and refreshes the view. The document you see is always the document that exists.

Every markdown file deserves to be read beautifully.
